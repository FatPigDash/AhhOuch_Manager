# AhhOuch 需求

### 目標使用者

1. 軟體名稱：AhhOuch
2. 使用者：多人使用，分為店家和消費者
3. 用於電腦和手機，預計用網頁為介面，減少硬體間的相容性問題(尤其是WINDOW和MAC)。
   1. 開發者的電腦當伺服器，知道網址的人才能查看。
   2. 每個使用者的資料是獨立的。

4. 開發資訊：
   1. 開發者不懂程式。
   2. 電腦：WIN11，系統語系為繁體中文、日文、英文。
   3. VS code

### 執行目標：一個用在養身館的軟體，目標使用者分為店家和客人。

#### 店家

1. 用來發布美容師資訊和班表，設定好內容後可以對社群軟體的群組發布及編輯內容。

2. 有美容師資訊和班表個別的獨立頁面。

3. 美容師資訊

   1. 每個美容師都有自己的資訊卡片，資訊卡片的名字由使用者自己輸入，可以是文字或是編號。
      1. 以卡片為主體，紀錄該美容師資訊，資訊有圖片和文字兩個欄位，說明如下
         1. 圖片可自行上傳或是貼上，但不強制要有圖片。
         2. 可以有多張圖片，並設定其中一張為封面圖。
         3. 文字說明分為完整介紹和簡短介紹兩種。

   1. 可直接把卡片資訊發布在社群軟體的群組，發布時可勾選要使用完整介紹還是簡短介紹。

4. 班表
   由上至下為標題、出勤欄位、出勤時段的區塊，完成後可發布在社群軟體的群組，

   1. 標題欄位為文字輸入框，由使用者自行輸入，可有多行文字。

   2. 出勤欄位：

      1. 列出資訊卡片的名字供點選當天有出勤的人員。
      2. 當點選出勤人員後，該人員的資訊會出現在出勤時段的欄位供編輯。

   3. 出勤時段：每個出勤人員獨立設定，分為顯示區和輸入區。輸入區的設定方式有兩種供選擇可以勾選。

      1. 手動輸入：
         1. 文字輸入框，旁邊有[+]的按鈕，輸入後點選[+]就會把該時間加入到顯示區。
         2. 採用24時制，輸入數字自動轉換(舉例：輸入「1830」自動轉換成「18:30」)。
      2. 自動換算：
         1. 使用者輸入上班時間，系統依照每班1.5小時的邏輯，列出上班開始12小時的班次。
         2. 每個班次都有核取方塊，勾選後該時段自動出現在顯示區。

   4. 點選發布後可選擇發布的程式，預設格式由上至下的說明如下：

      1. 最上方為標題列，依輸入內容呈現。

      2. 接著依出勤人員分別顯示：出勤人員編號、簡短介紹、出勤時段。舉例如下：

         #199 
         按摩力道足夠。特別擅長處理肩頸痠痛
         10:30、12:00、13:30、18:00

         #202
         按摩力道稍小，特別擅長處理大腿痠痛
         12:00、15:00、18:00

      3. 發布後將留存草稿，後續可依草稿進行修改後再發布。

   5. 修改草稿

      1. 點選要修改的草稿，修改的介面和編輯時一樣。

#### 客人

1. 用來記錄不同養身館美容師的資訊和心得。
2. 一間養身館一個頁面。
   1. 使用者可自行新增、編輯、刪除養身館資訊。
   2. 資訊有名稱、地址、幹部，只要填寫名稱就能建立頁面。
3. 每個養身館頁面內部用看板化管理(參考trello)
   1. 可自行增減看板及編輯看板名稱。
   2. 預設幾個看板，由左至右：觀望、體驗一次、已體驗超過一次、不會再體驗、已離職。
   3. 每個看板都能往下新增美容師卡片。
   4. 美容師卡片在看板中僅顯示標題和封面圖。
   5. 美容師卡片可以在同一個看板或是不同看板間拖曳，拖曳時有明確的插入符號。
   6. 美容師卡片可以設定預設排序或是手動排序，預設排序以卡片標題為基準，並且依照標準 Unicode 排序。
4. 美容師卡片
   1. 分為以下幾個區塊：標題、美容師簡介、心得。
   2. 標題由使用者自己輸入，可以是文字或是編號，只要輸入標題就能建立卡片。
   3. 美容師簡介：分為圖片、國籍、文字說明
      1. 整個美容師簡介可以收闔和開啟，預設開啟。
      2. 圖片可自行上傳或貼上，但不強制要有圖片。
      3. 可以有多張圖片，並設定其中一張為封面圖。
      4. 國籍為一個欄位，用文字輸入框輸入內容。
      5. 文字說明用文字輸入框輸入內容。
   4. 心得有以下區塊：日期、各項評分、文字說明
      1. 日期用圖像化日曆選取。
      2. 各項評分:
         1. 可設定多組模板，可以替每個模組編輯名稱。
         2. 每個模板可自行增加、編輯、刪除項目。
         3. 預設模板的項目：外表、身材、按摩、過程互動、服務、卸甲、回沖。
         4. 分數範圍為0~10分。
         5. 每個項目除了評分外也帶有文字輸入框可以補充說明。
         6. 可選擇要套用的模板，預設套用第一組模板。
      3. 文字說明用文字輸入框輸入內容。
      4. 可以新增多組心得
         1. 每一組心得的樣式都一樣。
         2. 新增時預設套用前一次的心得內容。



### 執行方向及架構

#### 開發順序

1. 客人的功能優先開發。
2. 電腦和手機的介面不同，預先保留兩種介面的架構，優先開發電腦介面。

#### 其他項目

1. 能夠打包成免安裝單一執行檔。
   1. 建立[打包輸出]的資料夾，把打包後的檔案放在[打包輸出]的資料夾。
   2. 自動刪除打包中產生的過程檔案。
2. 建立一個獨立檔案讓使用者修改該處即可變更軟體的名稱和版次(包含打包後的檔名)，而不是連動修改其他檔案的內容才會變更。
3. 建立一個軟體需求環境和套件的清單檔案，當終端輸入安裝環境與套件的指令後自動執行所有的安裝。
4. 建立README文字檔，須包含以下內容：
   1. 安裝環境與套件的指令。
   2. 測試階段的終端啟動指令、打包指令。
   3. 明確指出前述修軟體名稱和版次的檔案名稱。
   4. 說明軟體打包後的位置。
5. 建立[DATA]資料夾，把軟體的輸出檔案放在[DATA]資料夾裡面。
