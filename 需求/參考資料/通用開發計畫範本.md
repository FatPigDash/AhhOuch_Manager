# 〔專案名稱〕開發計畫

> 本檔為**通用開發計畫範本**。
> 開新專案時複製本檔，將 `〔…〕` 佔位符與《填寫說明》替換為實際內容，刪除用不到的章節。
>
> **使用方式**
> 1. 將標題改為實際專案名稱，刪掉本段「使用方式」。
> 2. 從頭逐節填寫；標示《PoC》《待確認》的項目可保留，邊開發邊回填。
> 3. `R 編號`對照表是驗收核心，務必逐條對應需求文件。
>
> **原則**：本範本只提供「結構與填寫方法」，**不預設任何功能或非功能需求**。
> 作業系統、語系、打包方式、輸出資料夾、設定檔慣例等，都屬於各專案的個別需求，
> 必須來自該專案的需求文件或與需求方確認後才填入（常見可確認項目見〔附錄 A〕）。

---

> 本文件依據 [`〔需求文件〕.md`](./〔需求文件〕.md) 撰寫，作為開發的總體規劃。

## 4. 系統架構與模組設計

### 建議專案目錄結構

> 以下為通用骨架，依實際介面形式（桌面/網頁/CLI）與需求增刪。
> 註：是否需要設定檔、輸出資料夾、安裝/打包腳本等，視該專案需求而定（見〔附錄 A〕），非預設。

```
〔專案〕/
├─ 〔進入點〕              # 程式進入點
├─ src/ 或 backend/        # 主程式
│  ├─ config.py            # 路徑常數、預設值
│  ├─ 〔介面層〕           # gui/ 或 routers/ 或 cli/
│  ├─ 〔核心邏輯〕         # core/ 或 services/
│  └─ utils/               # 共用工具
├─ tools/                  # （選用）實機探測小工具，供 PoC 驗證
└─ 〔其他依需求新增〕      # 設定檔、資料夾、腳本等（見附錄 A）
```

## 附錄 A：常見可確認項目（非預設，需逐案確認）
